Trestle

Trestle is a simulation and games engine that — from a content developer's perspective — is mainly focused on procedureal content.

From the engine developer's perspective, it also focuses on having a simple architecture and also letting the programmer gain more control over the graphics pipeline (by bypassing it entirely).

Needless to say, this has some intresting properties and requires more developer work, but is an intresting educational exprience.

Videos on YouTube

On the matter of educational expriences, there is a YouTube channel for Trestle engine.

Procedureal content

Note that "procedureal content" in this definition does not have to be randomly generated; it can be fixed form, it just needs to be generated procedureally. For example, if you want to place trees in a row evenly to make a level:

void trees() {
	for (float i = 0.0; i < 10.0; i += 1.0) {
		GenerateTree(vec3(-5.0, -1.0, i * 8.0));
	}
}

That counts as procedureal generation despite that it is completely deterministic and not pseudorandom.

Graphics

The traditional programmable graphics pipline is nice, but of course, it would be nice to be able to implement graphics completely programmably and to know what everything does. This is why we are trying to do graphics on compute (or GPGPU; think OpenCL or Vulkan Compute).

While we are aware that the fixed-function hardware used in the current modern pipeline is much faster and probably much better maintained than anything we could produce, we still think it is valuable to do this from an educational and flexibility persepctive.
